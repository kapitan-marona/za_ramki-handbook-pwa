# Storage Image Optimization

Дата: 2026-06-19

## Что найдено

Тяжёлые изображения инструкций сейчас находятся не в локальной папке проекта, а
в Supabase Storage bucket `kb-media`.

По выгрузке `sql/Migration/storage_objects.sql` самые тяжёлые файлы:

- `kb-articles/obmer/photo-1.png` - примерно 2.5 MB;
- `kb-articles/obmer/photo-2.png` - примерно 2.45 MB;
- `kb-articles/obmer/plan-1.png` - примерно 333 KB;
- `kb-articles/obmer/plan-2.png` - примерно 322 KB.

## Что уже сделано в приложении

В статьях добавлена ленивая загрузка картинок:

- первая картинка грузится сразу;
- остальные картинки грузятся только ближе к просмотру;
- декодирование картинок выполняется асинхронно.

Это улучшает ощущение скорости, но не уменьшает вес самих файлов.

## Как сжать сами изображения

1. В Supabase открыть Storage.
2. Открыть bucket `kb-media`.
3. Загрузить сжатые файлы рядом с оригиналами:
   - `kb-articles/obmer/photo-1.webp`;
   - `kb-articles/obmer/photo-2.webp`.
   - `kb-articles/obmer/plan-1.webp`;
   - `kb-articles/obmer/plan-2.webp`.
4. В markdown статьи заменить ссылки:
   - `photo-1.png` -> `photo-1.webp`;
   - `photo-2.png` -> `photo-2.webp`;
   - `plan-1.png` -> `plan-1.webp`;
   - `plan-2.png` -> `plan-2.webp`.
5. Проверить статью в приложении:
   - картинки открываются;
   - качество читаемое;
   - загрузка стала быстрее.

## Важно

Не удалять оригинальные PNG до проверки статьи. Лучше сначала загрузить WebP
рядом, обновить markdown статьи и только после smoke-test удалить старые PNG.
